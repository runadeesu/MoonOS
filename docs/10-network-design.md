# ネットワーク設計

## レイヤー構成

```
┌───────────────────────────────────────────┐
│ アプリケーション（ソケット API 経由）        │
├───────────────────────────────────────────┤
│ ソケット層 (libmoon が公開する BSD風API)     │
├───────────────────────────────────────────┤
│ トランスポート層 (TCP / UDP)                 │
├───────────────────────────────────────────┤
│ ネットワーク層 (IPv4 / IPv6, ICMP, ARP/NDP)   │
├───────────────────────────────────────────┤
│ リンク層 (Ethernet フレーミング)              │
├───────────────────────────────────────────┤
│ NIC ドライバ (virtio-net から開始, 実機NIC順次追加) │
└───────────────────────────────────────────┘
```

## 実装方針

- 標準規格（RFC）にのみ準拠して独自実装し、既存 TCP/IP スタックの
  コードは使用しません。
- 開発初期は QEMU の `virtio-net` デバイス向けドライバを実装し、
  実 NIC (Intel e1000, Realtek RTL81xx 等) のドライバは後続ステップで
  追加します。
- カーネル内蔵スタック（Linux 方式）とし、ユーザー空間からはソケット
  システムコール経由でアクセスします。将来的にユーザー空間ネットワーク
  スタック（DPDK 的な高速パス）をゲーム/低遅延用途向けオプションとして
  検討します。

## Moon Netd（ユーザー空間サービス, Step 7 以降）

- インターフェース管理（DHCP クライアント、静的 IP 設定）
- Wi-Fi 管理（スキャン・接続・WPA3 認証）— ドライバ経由でハードウェア制御
- ファイアウォール規則（パケットフィルタ、既定で受信拒否・送信許可）
- VPN / プロキシ設定の統合

## セキュリティ設計

- 既定でリスニングポートを持たない「セキュアバイデフォルト」構成
- アプリ毎のネットワークケーパビリティ（プロセス管理の Capability
  モデルと連携し、明示的な許可がないアプリはネットワークにアクセス不可）
- TLS 1.3 をシステム標準の暗号化ライブラリとして提供予定

## 実装ステップとの対応

Step 10（`docs/02-roadmap.md`）で以下の順に実装します。

1. virtio-net ドライバ + Ethernet フレーミング
2. ARP + IPv4 + ICMP（`ping` が通ることを最初のマイルストーンとする）
3. UDP → DHCP クライアント（DHCP で自動 IP 取得できることを確認）
4. TCP（3-way handshake, 再送制御, フロー制御）
5. ソケット API を `libmoon` 経由でユーザー空間に公開
6. IPv6, TLS 1.3, `netd` サービス化
